# 年度 Top20 损失金额及损失率产品分析

## 项目概述

将制造业品质管理中依赖 Excel、查找函数和数据透视表的月度分析流程，重构为可复用的事实表、SQL 指标层和 Tableau 看板数据源。项目关注成本损失与质量改善两个不同维度，支持产品、工序和月度的异常定位。

项目基于真实业务流程抽象，公开版本仅使用脱敏/模拟数据复现业务逻辑。金额、排名和分析结论均不代表原公司的真实经营数据。

## 业务问题

1. 每月哪些产品的不良损失金额最高，应优先核查成本损失？
2. 哪些产品的综合不良率最高，应优先进入质量改善清单？
3. 高损失产品主要由哪一道工序贡献？
4. 哪些产品存在连续 Top20、环比上升或数据质量异常？

## 数据建模与指标

- **事实表粒度**：`月份 + 产品 + 工序`，共 18,000 行。
- **工序损失金额**：`loss_qty × unit_price`；有损失数量但单价为 0 的记录单独标记为数据质量问题。
- **损失金额 Top20**：按产品月 `total_loss_amount` 降序，服务于成本损失优先级。
- **综合不良率 Top20**：按产品月 `composite_defect_rate` 降序，服务于质量改善优先级；不能与损失金额 Top20 混用。
- **综合不良率**：`1 - (1 - 铸造不良率) × (1 - 机加不良率) × (1 - 抛光不良率)`，避免把多个工序的生产分母直接相加。

## 分析链路

1. 用 pandas 清洗不良数量、生产数量和工序单价，并将宽表转成长表。
2. 构建产品月工序级事实表，统一字段与粒度。
3. 用 MySQL 生成产品月、工序月、Top20、环比与数据质量检查指标。
4. 输出 Tableau 数据源，按产品、工序和月份观察问题优先级。

## 分析边界
